Feldcodes einfügen

Im Dokumenteneditor können Sie Felder mit dynamischem Inhalt hinzufügen, der von Ihren Systeminformationen oder anderen Inhalten abhängt. Bitte verwechseln Sie Felder nicht mit Inhaltssteuerelementen, da es sich um zwei verschiedene Funktionen handelt.

Um ein Feld einzufügen,

  1. Gehen Sie zur Registerkarte Einfügen.
  2. Klicken Sie auf die Schaltfläche Feld.
  3. Geben Sie im geöffneten Dialogfenster die anzuzeigenden Feldfunktion ein und klicken Sie auf OK.

Die Feldfunktionen sind folgende:

ONLYOFFICE Feldfunktion Beschreibung Syntax Schalter
MERGEFIELD Der Code fügt einen Link zu einem Datenfeld aus der anderen Quelle in das aktuelle Dokument ein. Beim Zusammenführen ersetzen Daten aus der anderen Quelle den aktuellen Feldwert. { MERGEFIELD FieldName [ Schalter ]}
  1. \b - gibt den Text an, der vor dem MergeField-Feld eingefügt werden soll, wenn dieses nicht leer ist.
  2. \f - gibt den Text an, der nach dem MergeField-Feld eingefügt werden soll, wenn dieses nicht leer ist.
  3. \m - gibt an, dass das MergeField-Feld ein zugeordnetes Feld ist.
  4. \v - ermöglicht die Zeichenkonvertierung für die vertikale Formatierung.
PAGENUM Der Code fügt die aktuelle Seitenzahl ein.
PAGECOUNT Der Code fügt die Gesamtzahl der Seiten im aktuellen Dokument ein.
FORMTEXT
TOC Der Code erstellt ein Inhaltsverzeichnis. Das Inhaltsverzeichnis basiert auf den angewendeten Überschriftenstilen, Gliederungsebenen, Bildunterschriften usw., die in den Inhaltsverzeichniseintragsfeldern angegeben sind. { TOC [ Schalter ] }
  1. \a - erstellt eine Liste von Objekten, die keine Beschriftungen und Zahlen enthält.
  2. \b - erstellt ein Inhaltsverzeichnis für den mit Lesezeichen versehenen Teil des Dokuments.
  3. \с - erstellt eine Liste von Objekten des angegebenen Typs.
  4. \d - legt den Trennzeichentyp für Seitenzahlen fest.
  5. \f - erstellt ein Inhaltsverzeichnis basierend auf den Inhaltsverzeichnisfeldern und nicht auf den Dokumentstrukturebenen.
  6. \h - wendet die Hyperlink-Formatierung auf das Inhaltsverzeichnis an.
  7. \l - gibt die erforderlichen Ebenen der Inhaltsverzeichnisfelder an. Die Ebenen, die nicht im angegebenen Bereich enthalten sind, werden weggelassen.
  8. \n - erstellt ein Inhaltsverzeichnis entweder vollständig oder nur mit den Elementen aus dem angegebenen Bereich (z. B. „1-9“) und lässt die Seitenzahlen weg.
  9. \o - erstellt ein Inhaltsverzeichnis, das nur die Strukturebenen des Dokuments und nicht die Inhaltsverzeichnisfelder verwendet.
  10. \p - gibt das Trennzeichen zwischen dem Element und der entsprechenden Seitenzahl an.
  11. \s - fügt der Seitenzahl eine fortlaufende Nummer hinzu.
  12. \t - erstellt ein Inhaltsverzeichnis, das nur benutzerdefinierte Stile anstelle der Standard-Stile verwendet.
  13. \u - erstellt ein Inhaltsverzeichnis, wobei nur die angewendete Gliederungsebene der Absatzformatierung verwendet wird.
  14. \w - behält die Tabellierung in Tabelleneinträgen bei.
  15. \x - behält Zeilenumbrüche in Tabelleneinträgen bei.
  16. \z - blendet die Seitenzahl aus, während das Inhaltsverzeichnis als Webdokument-Layout angezeigt wird.
PAGEREF Der Code fügt die Seitenzahl als Lesezeichen zum Querverweis ein. { PAGEREF Lesezeichen [\* Formatschalter ] }
  1. \h - erstellt einen Hyperlink zum angegebenen Absatz.
  2. \p - fügt die Position des angegebenen Absatzes relativ zum Lesezeichen ein.
PAGE Der Code fügt die Nummer der aktuellen Seite ein. { PAGE [\* Formatschalter ] }
NUMPAGES Der Code fügt die Gesamtzahl der Seiten im Dokument ein. { NUMPAGES }
ASK Der Code fügt ein Feld ein, in dem der Benutzer aufgefordert wird, Daten einzugeben, die dann als Lesezeichen gespeichert werden. Der Editor fordert den Benutzer jedes Mal auf, wenn dieses Feld aktualisiert wird oder während der Zusammenführung. { ASK Lesezeichen"Aufforderung" [ Optionale Schalter ] }
  1. \d - fügt den Standard-Ersetzungstext als Lesezeichen ein.
  2. \o - beim Zusammenführen von Dokumenten wird nur einmal zur Eingabe aufgefordert.
REF Der Code fügt den Inhalt des Lesezeichens ein. Das Lesezeichen sollte in das Dokument eingefügt werden. { [REF] Lesezeichen [ Schalter ] }
  1. \d - gibt das Trennzeichen für die Seitenzahl an.
  2. \f - erhöht Fußnoten- oder Notizennummern und fügt sie in das Dokument ein.
  3. \h - erstellt einen Hyperlink zum angegebenen Absatz.
  4. \n - fügt Absatznummern hinzu, falls vorhanden.
  5. \p - fügt die Position des angegebenen Absatzes relativ zum Lesezeichen ein.
  6. \r - fügt die relative Nummer des angegebenen Absatzes ein.
  7. \t - lässt Symbole weg, die nicht numerisch sind oder kein Trennzeichen enthalten.
  8. \w - fügt die vollständige Nummer des angegebenen Absatzes ein.
HYPERLINK Der Code fügt einen Hyperlink zu einem Element im Dokument, zu einem Dokument auf Ihrer Festplatte, einem Portal oder einer Drittanbieterressource ein. { HYPERLINK "Dateiname" [ Schalter ] }
  1. \l - gibt das Element im Dokument an, zu dem es gewechselt werden soll.
  2. \m - gibt die Koordinaten eines Hyperlinks für eine serverseitige Imagemap an.
  3. \n - öffnet das Enddokument in einem neuen Fenster.
  4. \o - gibt den QuickInfo-Text an.
  5. \t - gibt das Ziel des Hyperlinks an.
TIME Der Code fügt die aktuelle Uhrzeit basierend auf dem System des Benutzers in das Dokument ein. { TIME [\@ "Date-Time Picture"] }
DATE Der Code fügt das aktuelle Datum basierend auf dem System des Benutzers in das Dokument ein. { DATE [ \@ "Date-Time Picture"] [Switches] }
FORMULA Der Code führt die Berechnung gemäß der eingegebenen Formel durch. { = Formula [ Lesezeichen ] [\#Numeric Picture ] }
SEQ Der Code bietet eine Möglichkeit, Kapitel, Tabellen, Abbildungen und andere Elemente in einem Dokument der Reihe nach zu nummerieren. Wenn Sie ein Element zusammen mit seinem Seq-Feld hinzufügen, entfernen oder verschieben, können Sie die anderen Seq-Felder im Dokument aktualisieren, um die überarbeitete Reihenfolge anzuzeigen. { SEQ Identifier [ Lesezeichen ] [ Schalter ] }
  1. \c - fügt die nächstgelegene vorhergehende Sequenznummer ein.
  2. \h - blendet das Feldergebnis aus (wird für Querverweise verwendet).
  3. \n - fügt die nächste Sequenznummer ein.
  4. \r - die Nummerierung beginnt mit der angegebenen Zahl.
  5. \s - die Nummerierung beginnt mit der angegebenen Gliederungsebene.
STYLEREF Der Code fügt Text hinzu, der mit einem bestimmten Stil formatiert wurde. Wenn er in einer Kopf- oder Fußzeile platziert wird, zeigt er die erste oder letzte Instanz von mit diesem Stil formatiertem Text aus dem Hauptinhalt der aktuellen Seite an und ermöglicht so die Erstellung von Kopf- oder Fußzeilen im Wörterbuchstil. { STYLEREF StyleIdentifier [ Schalter ] }
  1. \l - startet die Suche am Ende der aktuellen Seite.
  2. \n - fügt die Nummerierung ab dem angegebenen Absatz ein.
  3. \p - fügt die Position des angegebenen Absatzes relativ zum Lesezeichen ein.
  4. \r - fügt die relative Nummer des angegebenen Absatzes ein.
  5. \t - lässt Symbole weg, die nicht numerisch sind oder kein Trennzeichen enthalten.
  6. \w - fügt die vollständige Nummer des angegebenen Absatzes ein.
NOTEREF Der Code fügt eine Fußnoten- oder Endnotenverweismarke ein, die mit einem Lesezeichen versehen ist, um auf dieselbe Notiz zu verweisen oder Fußnoten und Endnoten miteinander zu vergleichen. Wenn die Reihenfolge der Notizen geändert wird, spiegelt das Ergebnis des Felds die neue Nummerierung wider. { NOTEREF Lesezeichen [ Schalter ] }
  1. \f - fügt ein mit dem Fußnoten-Referenzstil oder dem Endnoten-Referenzstil formatiertes Referenzzeichen ein.
  2. \h - erstellt einen Hyperlink zur angegebenen Fuß- oder Endnote.
  3. \p - fügt die relative Position der Fuß- oder Endnote ein.
